## 🔧 Step 1: Push Your Code to GitHub

### 1a. Create a GitHub Repository

1. Go to [github.com/new](https://github.com/new)
2. Repository name: `ecommerce-app`
3. **Private or Public**: Choose Public (Vercel/Railway work better with public)
4. **Initialize**: Leave unchecked
5. Click **Create repository**

### 1b. Push Your Local Code

In your project folder, run:

```powershell
# Initialize git (if not already done)
git init
git add .
git commit -m "Initial commit: Full-stack eCommerce with AI"

# Add remote and push
git remote add origin https://github.com/YOUR_USERNAME/ecommerce-app.git
git branch -M main
git push -u origin main
```

> **Replace `YOUR_USERNAME`** with your actual GitHub username

---

## 🚂 Step 2: Deploy Backend to Railway

Railway automatically deploys when you push to GitHub.

### 2a. Create Railway Project

1. Sign up at [railway.app](https://railway.app)
2. Click **New Project**
3. Select **Deploy from GitHub repo**
4. Click **Connect GitHub** and authorize
5. Select your `ecommerce-app` repository
6. Select `backend` directory as root directory
7. Click **Deploy**

Railway will:
- Build your Docker image
- Create PostgreSQL database automatically
- Deploy your API
- Provide a public URL: `https://ecommerce-app-production.up.railway.app`

### 2b. Configure Environment Variables

1. In Railway Dashboard → Your Project → **Variables**
2. Add these variables:

```
DB_HOST=postgres
DB_PORT=5432
DB_NAME=railway
DB_USER=postgres
DB_PASSWORD=<auto-generated>
DB_DIALECT=postgres

PORT=5000
NODE_ENV=production
JWT_SECRET=<generate-a-strong-random-string>

STRIPE_SECRET_KEY=<your-stripe-key>
STRIPE_PUBLISHABLE_KEY=<your-stripe-key>
OPENAI_API_KEY=<your-openai-key>

REDIS_URL=redis://localhost:6379

APP_URL=<your-railway-backend-url>
FRONTEND_URL=<your-vercel-frontend-url>
```

> **Note**: Railway automatically creates PostgreSQL and provides connection strings

3. Click **Save**

---

## ▲ Step 3: Deploy Frontend to Vercel

### 3a. Deploy on Vercel

1. Go to [vercel.com/import](https://vercel.com/import)
2. Click **Import Git Repository**
3. Paste your GitHub repo URL: `https://github.com/YOUR_USERNAME/ecommerce-app`
4. Click **Continue**
5. Select **Root Directory**: `frontend`
6. Click **Deploy**

Vercel will:
- Build your React app
- Deploy to CDN
- Provide URL: `https://ecommerce-app-yourname.vercel.app`

### 3b. Configure Environment Variables (Vercel)

1. In Vercel Dashboard → Your Project → **Settings** → **Environment Variables**
2. Add:

```
REACT_APP_API_BASE_URL=https://your-railway-backend-url/api
REACT_APP_STRIPE_PUBLIC_KEY=pk_test_...
REACT_APP_ENVIRONMENT=production
```

3. Click **Save**

## 🔗 Update Your API Base URL

After deployment, update your frontend's API base URL:

**In `frontend/.env.production`:**
```
REACT_APP_API_BASE_URL=https://your-ecommerce-app-production-up-railway-app.com/api
```

Then push to GitHub:
```powershell
git add .
git commit -m "Update API URL for production"
git push
```

Vercel will auto-redeploy ✅

## 🆘 Troubleshooting

### Frontend Can't Connect to API
**Problem**: `CORS error` or `Cannot reach API`

**Solution**:
1. Check `REACT_APP_API_BASE_URL` in Vercel env vars
2. Verify backend URL is updated
3. Check Railway logs for errors
4. Redeploy frontend: `git push`

### Database Connection Error
**Problem**: `Cannot connect to database`

**Solution**:
1. Railway auto-creates PostgreSQL
2. Check Railway Dashboard → PostgreSQL plugin
3. Verify `DB_HOST=postgres` (not localhost)
4. Check environment variables are set

### SSL Certificate Error
**Problem**: `SSL certificate error`

**Solution**: Railway provides HTTPS automatically. This usually resolves in 10 minutes.

### Out of Storage
**Problem**: `Disk full on Railway`

**Solution**:
1. Check large files in git history: `git log --all --full-history -- file`
2. Clean git history (advanced): `git filter-branch`
3. Or upgrade to paid plan
